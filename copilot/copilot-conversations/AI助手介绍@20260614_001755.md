# CAN 通讯

## 1. 物理层

### 1.1 硬件连接
![[Pasted image 20260607234748.png]]

- **组成**：MCU → CAN 控制器 → CAN 收发器  
- **总线**：CAN_H、CAN_L（双绞线，提高抗干扰能力）  
- **终端电阻**：网络两端必须各接 **120Ω** 电阻（匹配电缆的 [[传输线]] 特性阻抗，模拟无穷远传输线）

### 1.2 电平信号（ISO 11898 标准）

| 名称 | 性质 | 逻辑 | CAN_H | CAN_L | 压差 |
|------|------|------|-------|-------|------|
| recessive（隐性） | 隐形 | 1    | 2.5V  | 2.5V  | 0V   |
| dominant（显性）  | 显性 | 0    | 3.5V  | 1.5V  | 2V   |

- **隐性**：总线空闲或发送逻辑 1  
- **显性**：发送逻辑 0，两条线压差 2V，可覆盖隐性  
- 多节点同时发送时，通过 **[[总线仲裁]]** 解决冲突（显性优先）

## 2. 协议层 — 帧结构

CAN 有 5 种帧类型，其中硬件自动完成错误帧、过载帧、帧间隔：

| 帧类型 | 用途 |
|--------|------|
| 数据帧 | 发送单元向接收单元传送数据 |
| 遥控帧 | 请求具有相同 ID 的单元发送数据 |
| 错误帧 | 检测到错误时通知其他单元 |
| 过载帧 | 通知其他节点暂缓发送（本节点忙） |
| 帧间隔 | 分隔帧 |

### 2.1 数据帧 / 遥控帧格式
两种帧均有 **标准格式（11 位 ID）** 和 **扩展格式（29 位 ID）**，整体结构相似，只是在仲裁段和控制段做扩展。

![[Pasted image 20260608001358.png]]
![[Pasted image 20260608001406.png]]

#### 帧组成

| 段名 | 描述 |
|------|------|
| 帧起始 (SOF)       | 1 bit 显性电平 |
| 仲裁段 (Arbitration)| 标识符 + RTR 位，决定优先级 |
| 控制段 (Control)   | IDE + r0/r1 + DLC (4位数据长度) |
| 数据段 (Data)      | 0 ~ 8 字节 |
| CRC 段             | 传输校验 |
| ACK 段             | 接收确认 |
| 帧结束 (EOF)       | 7 个隐性位 |

#### 仲裁段详解

- 标准格式：11 位 ID + RTR  
- 扩展格式：11 位 ID + SRR (隐性) + IDE (隐性) + 18 位 ID + RTR  
- **显性（0）优先级更高**：标准帧 IDE 为显性，因此在总线上标准帧优先级始终高于扩展帧。  
- 每个节点通过 **[[过滤器]]** 筛选只接收需要的 ID。

#### 控制段

- 标准帧：IDE + r0 + DLC (4 bits)  
- 扩展帧：r1 + r0 + DLC (4 bits)  
- IDE：0 = 标准帧，1 = 扩展帧  
- r0、r1：保留位，发送必须显性，接收不关心  
- DLC：指示数据段字节数（0 ~ 8）

## 3. CAN 外设（以 STM32F103 为例）

### 3.1 内部结构
![[Pasted image 20260611211434.png]]

- **位速率**：最高 1 Mbit/s  
- **帧格式**：支持 11 位标准帧 和 29 位扩展帧  
- **发送**：3 个发送邮箱  
- **接收**：2 个 FIFO（三级缓冲），每个 FIFO 配有可编程滤波器组

### 3.2 波特率计算

$$
\text{波特率} = \frac{ f_{\text{APB}} }{ \text{Prescaler} \times ( \text{TBS1} + \text{TBS2} + \text{SJW} ) }
$$

- TBS1：位段 1 的时间单元数  
- TBS2：位段 2 的时间单元数  
- SJW：再同步跳转宽度

## 4. 软件配置

### 4.1 初始化流程

1. 使能 CAN 外设时钟  
2. 调用 `CAN_Init()` 配置工作模式、波特率等  
3. 配置过滤器（`CAN_FilterInit()`）  
4. 准备消息结构体（`CanTxMsg` / `CanRxMsg`）  
5. 启动 CAN 模块

### 4.2 过滤器配置

过滤器决定哪些报文能被接收，保存在 `CAN_FilterTypeDef` 结构体中：

```c
typedef struct {
  uint32_t FilterIdHigh;
  uint32_t FilterIdLow;
  uint32_t FilterMaskIdHigh;
  uint32_t FilterMaskIdLow;
  uint32_t FilterFIFOAssignment;  // 指定放入 FIFO0 或 FIFO1
  uint32_t FilterBank;            // 过滤器组编号 0 ~ 13
  uint32_t FilterMode;            // ID 掩码模式 / ID 列表模式
  uint32_t FilterScale;           // 32 位 / 16 位
  uint32_t FilterActivation;      // 使能
  uint32_t SlaveStartFilterBank;  // 双 CAN 时使用
} CAN_FilterTypeDef;
```

#### 过滤器模式

| 模式 | 说明 |
|------|------|
| `CAN_FilterMode_IdMask` | ID 掩码模式：根据参考 ID 和屏蔽码，只关心被屏蔽码标记的位 |
| `CAN_FilterMode_IdList` | ID 列表模式：必须与列表中某个 ID **完全匹配**才接收 |

- **ID 掩码模式示例**  
  参考 ID = `0x123` (1 0010 0011)  
  屏蔽码 = `0x7F0` (111 1111 0000)  
  匹配规则：高 7 位必须与参考 ID 相同，低 4 位任意。

- **ID 列表模式**：  
  `FilterMaskIdHigh/Low` 被用作额外的 ID 寄存器，32 位模式可存 2 个 ID，16 位模式可存 4 个 ID。

#### 过滤器尺度

| 尺度 | 说明 |
|------|------|
| `CAN_FilterScale_32bit` | 使用一个 32 位过滤器 |
| `CAN_FilterScale_16bit` | 使用两个 16 位过滤器 |

#### ID 存放规则（标准帧）

- 标准帧 11 位 ID 应存放在 `FilterIdHigh[15:5]`，低位部分用于扩展 ID。  
- 屏蔽码通常设置为 `0xFFE0`（即只检查 11 位 ID）。

#### 过滤器组与 FIFO 对应

STM32F103 共有 14 个过滤器组（0 ~ 13），每个组可配置将接收到的报文路由到 FIFO0 或 FIFO1：

```
CAN 总线
   │
   ▼
CAN 控制器
   ┌────────────┐
   │ Filter 0   ───► FIFO0
   ├────────────┤
   │ Filter 1   ───► FIFO1
   ├────────────┤
   │ ...        │
   ├────────────┤
   │ Filter 13  ───► FIFO0 或 FIFO1
   └────────────┘
        │
        ▼
     CPU 读取
```

## 5. 发送与接收流程

**发送**  
1. CPU 填充发送邮箱（ID、DLC、数据）  
2. CAN 控制器将报文发送到总线  
3. 总线仲裁后优先发送高优先级报文  

**接收**  
1. 报文到达目标节点  
2. 硬件过滤器筛选 ID  
3. 匹配的报文存入对应的 FIFO  
4. CPU 从 FIFO 读取数据  

---

## 6. 总结

- CAN 总线通过 **差分信号** 保证抗干扰能力（CAN_H/CAN_L 双绞线）。  
- 仲裁基于 **显性优先**（线与逻辑），报文 ID 越小优先级越高。  
- 过滤器可灵活屏蔽或指定需要接收的 ID，提高效率。  
- 波特率由 APB 时钟分频及 **位时间** 参数共同决定。
